Introduction

Text Adventure is a text-based RPG set in a fantasy world. You will be the hero of our story, are you ready for this exciting adventure?

Game Plot

In a distant era, in the lands of Margorgh, in the darkest depths of a shadowy forest, it is said that there exists a Castle ruled by the forces of evil, housing an unimaginable power. For millennia, humanity has sought to wrest this power from the forces of evil; battles and wars have marked the quest for this power.
Over time, the traces of the Castle and the power within have been lost, vanished like leaves on trees in winter, or at least, so it was believed... A mysterious scroll was discovered by a warrior with a golden heart, a scroll containing knowledge that he could not have imagined and that would lead him to an unbelievable destiny....


Manual

Gameplay

The game is set in two different levels, one located in a forest, the latter in a castle.
Each one of them is subdivided in various section, that we will call "room" for simplicity.

At the first launch of the game, the program will load the main menu, where you can choose between:

  • start a new game
  • load a pre-existing game (if existing)
  • remove a pre-existing game (if existing)
  • see a list of the pre-saved games

If you start a new game, you will be asked to enter your hero-name. In any moment of the game you can use help to request the list of the available commands. [...] The goal of the game is to beat the final boss!

Graphic Interface

For the GUI it has been used JavaFX FXML, with which you can implement the graphics of a desktop application. The graphic interface is composed in three main parts:

  • an "artistic" section, where there is a image/video (created and rendered with the program Blender) which represents the content of that specific room
  • an output section, where you can see the program's response
  • an input section, with which you can interact with the game

How To Play

The user interact with the display using a custom command line, which is located at the bottom of the game window.
The user have to type the command followed by (if any) arguments and then press enter to trigger the specific in-game action.
Example:

>>command <argument>

The list of all the possible commands is available below

Commands List

Name Argument Description
newGame - Create a new game.
It can be utilized only in the menu.
loadGame Name of the saved game Command for loading a game previously saved.
save (Optional) name of the save Command for save the current game.
Require a name for the game that you are saving.
deleteGame Name of the save to be deleted Command for deleting a previously saved game.
Enter the name of the game to be deleted
listGames - Command that showing a list of the saved games.
show (Optional) one between inventory / stats Command for showing the stats and inventory of the player.
show inventory for showing only the player's inventory.
show stats the showing only the player's stats.
move Cardinal directions:
north / south / east / west		 Command manage the movement in the map using cardinal directions.
If possible you move in the digit direction
look - Command for showing a list of the items in the room.
pick Name of the item to be taken Command for taking an item from the items present in the room
Return error message if the inventory has not enough space.
fight - Command for starting a battle with a hostile mob (enemy).
attack - Command for attacking a monster in a fight.
dodge - Command for dodging an incoming attack from a monster.
It can be used 3 times per fight.
use Name of the item to be used in the inventory Command for using an item in the player's inventory.
wai - Command for showing the properties (locked/unlocked) of directions that can be used with move
help - Show all the possible commands in a precise moment of the game.
clear - Command for cleaning the game terminal.
